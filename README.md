# Link Index

個人で運用する、業務用リンクの索引アプリ。ビルド不要・サーバー不要の単一 HTML ファイルで動作する。

業務で参照するドキュメント・管理画面・社内ツールのリンクを、検索 / タグ / 工程別グループで一元管理する。Chrome のタブやブックマークが肥大化して探しにくくなる問題を、「索引(index)」として解決することを狙う。

## 特徴

- **単一ファイル** — `index.html` ひとつで完結。ビルドもサーバーも不要
- **クライアント完結** — データはブラウザの localStorage にのみ保存。外部送信・外部 fetch を一切行わない
- **2 つのビュー** — 素早く開くための `Board` と、確認・編集向けの `List` を切り替え
- **Board** — グループごとにタイトルのみのボタンを並べ、1 クリックで新規タブで開く。`Pinned` をボード上部に、全リンクをまとめた `All` セクションを最下部に表示
- **全文検索 / タグ絞り込み** — title / URL / group / tag / note を対象に入力即時で絞り込み。一致箇所はハイライト
- **タイトル自動入力** — URL の貼り付け時に title 候補を自動推定（SharePoint の `file` / `id` / `RootFolder` / `sourcedoc` などにも対応）
- **グループ管理** — 工程別グループの追加・リネーム・削除、ボード上での折りたたみ
- **グループ色** — グループ管理モーダルでプリセット 9 色から色を設定でき、Board のグループセクションに左ボーダーとカラードットで反映される
- **手動並び替え** — Board のカードをドラッグ&ドロップでグループ内・Pinned 内で並び替え。グループの表示順もグループ管理モーダルのドラッグで変更できる（検索・タグ絞り込み中と All セクションは対象外）
- **未分類は Inbox 扱い** — グループ未指定のリンクは List / All セクションの先頭に登録が新しい順で表示され、仮登録したリンクをすぐ見つけてグループ設定できる
- **Import / Export** — 全データを JSON で書き出し / 取り込み（マージ・置換）
- **レスポンシブ** — デスクトップ・モバイル両対応
- **外部依存ゼロ** — フォントは system フォントを使用し、CDN を含む外部リソースを読み込まない

## 使い方

ローカルで開くだけで動作する。

```bash
open index.html
```

`file://` でもほとんどの機能が動くが、クリップボード API などブラウザによっては HTTP 配信が必要な機能もあるため、確認時はローカルサーバー経由を推奨する。

```bash
python3 -m http.server 8000
# http://localhost:8000 を開く
```

初回起動時は localStorage が空のため、サンプルのリンクとグループが表示される。`Settings ▸ Data` から `Clear All` で全削除、`Restore Samples` でサンプルへの復元ができ、現在の localStorage 使用量も確認できる。

## デプロイ

静的ファイルのみのため、任意の静的ホスティングに `index.html` と `favicon.svg` を配置すれば動作する。

- **Cloudflare Pages** / **GitHub Pages** などに Git push トリガで自動デプロイ
- 私用 PC で改修 → push → デプロイ → 職場 PC で利用、というサイクルを想定

## キーボードショートカット

| キー                     | 動作                                           |
| ------------------------ | ---------------------------------------------- |
| `Cmd`/`Ctrl`+`V`         | URL を貼り付けてクイック追加バーを起動         |
| `Cmd`/`Ctrl`+`Shift`+`L` | クイック追加バーを開く（貼り付け待ち）         |
| `/`                      | 検索ボックスにフォーカス                       |
| `n`                      | 新規リンク追加モーダルを開く                   |
| `?`                      | 使い方とショートカット（Help）を開く           |
| `Esc`                    | クイック追加バー / モーダル / メニューを閉じる |
| `Cmd`/`Ctrl`+`Enter`     | リンク / グループの追加・編集フォームで登録    |

グループ / タグの入力欄では `↑` `↓` で候補を移動、`Enter` で確定、`Tab` で次の項目へ移動できる。

これらの操作は、`Settings`（歯車）メニューの `Help`（または `?` キー）で開く **Help** ダイアログからも画面上で確認できる。

### クイック追加

リンク登録のハードルを最小化するための簡易登録。コピーした URL を画面上で `Cmd`/`Ctrl`+`V` すると、画面上部にクイック追加バーが開き、URL とタイトル候補（URL から自動推定）が入った状態になる。あとは `Enter` を押すだけで登録される（グループは未指定のまま）。

- バーには `Add ↵` と `Details →` のキーヒントが表示される
- タイトルは編集可
- グループ・タグ・メモは `→`（入力末尾）か `Details` でフォームへ（URL / タイトルは引き継がれる）
- 空のまま開くには `Cmd`/`Ctrl`+`Shift`+`L`。フォーカスされたバーに `Cmd`/`Ctrl`+`V` で貼り付ける
- URL 以外の内容を貼り付けたときはバーを開かず、通常の貼り付けとして扱う
- 同じ URL が登録済みなら重複登録せず通知する

## データ

ブラウザの localStorage に次の形式で保存する。

- **Key**: `link-index:v1`
- **Value**: `{ version, groups, groupColors, collapsedGroups, links }`

`groupColors` はグループ名からプリセット色 ID（`gray` / `brown` / `orange` / `yellow` / `green` / `blue` / `purple` / `pink` / `red`）へのマップで、色未設定のグループはキー自体を持たない。

各リンク(`Link`)は次のフィールドを持つ。

| フィールド     | 型         | 説明                                        |
| -------------- | ---------- | ------------------------------------------- |
| `id`           | `string`   | 内部 ID（UUID）                             |
| `title`        | `string`   | 必須                                        |
| `url`          | `string`   | 必須。`http(s)://` で始まる                 |
| `group`        | `string`   | 工程・用途のまとまり。未指定時は `no group` |
| `tags`         | `string[]` | 小文字・trim 済み・重複除外                 |
| `note`         | `string`   | 任意                                        |
| `createdAt`    | `number`   | 追加日時（unix ms）                         |
| `updatedAt`    | `number?`  | 編集時に付与                                |
| `pinned`       | `boolean`  | `Pinned` セクションに固定表示するか         |
| `lastOpenedAt` | `number?`  | 最後にリンクを開いた日時（unix ms）         |
| `order`        | `number?`  | グループ内の手動並び順。未設定は末尾扱い    |
| `pinnedOrder`  | `number?`  | `Pinned` 内の手動並び順。未設定は末尾扱い   |

### Import / Export

`Settings`（歯車アイコン）メニューから操作する。

- **Export** — 全リンクを JSON ファイルでダウンロード
- **Import (Merge)** — 既存データに追加。URL が重複するエントリはスキップ
- **Import (Replace)** — 既存データを全置換

職場 PC ↔ 私用 PC の同期はこの Export / Import で行う（自動同期は持たない）。localStorage を消去するとデータは失われるため、Export でバックアップを取る運用を前提とする。保存容量の上限に達した場合はトーストで警告するので、Export でバックアップする。

## 技術スタック

| 領域         | 採用技術                                              |
| ------------ | ----------------------------------------------------- |
| 言語         | HTML / CSS / Vanilla JavaScript（フレームワークなし） |
| 永続化       | Web Storage API (localStorage)                        |
| フォント     | system フォントスタック（外部フォント読み込みなし）   |
| ホスティング | Cloudflare Pages / GitHub Pages など静的ホスティング  |
| ビルド       | 不要                                                  |

## セキュリティ / プライバシー

- サーバーサイド処理を一切行わない（静的ファイルのみ）
- データは利用者の localStorage に閉じ、外部送信しない
- 認証情報・API トークン等は扱わない
- リンク先 HTML を取得する外部 metadata fetch は行わない。title 推定はクリップボード情報または URL 文字列からのみ行う
- 外部依存（CDN 等）を持たないため、業務ネットワークの制約下でも表示が崩れにくい

改修方針・アーキテクチャ・規約は [AGENTS.md](AGENTS.md) を参照。仕様の一次情報は実装（`index.html`）そのもの。

### 仕様メモ

- **最近開いたリンクは表示しない** — `lastOpenedAt` は内部的に記録するが、Board への「最近開いたリンク」表示は提供しない（仕様上の決定。`Pinned` セクションのみ表示）
- **note は編集ダイアログのみ** — note は Board / List には表示せず、確認・編集は追加/編集ダイアログで行う（全文検索の対象には含まれる）
- **フォント** — system フォントスタックのみを使用し、Google Fonts などの外部フォントは読み込まない（外部依存ゼロを優先した設計判断）
